## 2026-07-03

- モバイル初回ロード時にグラフ（`#lineChart`）が幅61px程度で潰れる/空になるバグを修正した。
  - 原因：`drawStackedBarChart` / `drawWithdrawChart` / `drawPieChart` が `chart.clientWidth` を参照するが、初回はレイアウト確定前（clientWidthが未確定な小さい値）に呼ばれていた。
  - 対策1：3つの描画関数の先頭に、`clientWidth`/`clientHeight` が閾値未満（chart系80px未満、pie系40px未満）のときは描画をスキップして直前の canvas 状態を保持するガードを追加。
  - 対策2：`chart.parentElement` と `pieChart.parentElement` に `ResizeObserver` を付け、サイズ確定・変化のたびに `update()` を呼んで再描画するようにした（画面回転にも自動対応）。
  - 対策3：`window.addEventListener("load", ...)` と初期化直後の `requestAnimationFrame(update)` を追加し、レイアウト確定直後にもう一度描画するフォールバックを入れた。
  - ブラウザで、chart-panel の幅を一時的に20pxまで縮めて `update()` を呼んでも canvas の描画属性が壊れないこと（ガードが機能すること）、幅を戻すと ResizeObserver 経由で正しい解像度に再描画されること（clientWidth×dprとcanvas.widthが一致）を、積立・取り崩し両モードで確認した。
- 年次推移テーブルに「→ 横にスクロールできます」の注記（`.scroll-hint`）を追加した。画面幅680px以下でのみ表示されるようCSSメディアクエリで制御。デスクトップ幅では非表示であることをブラウザで確認済み（実機での680px以下表示は、CSSルールが正しく登録されていること・`matchMedia`条件式の妥当性・要素内容と配置を検証して代替確認した。ローカル環境の制約でウィンドウ実寸を680px以下に変更してのスクリーンショット確認はできなかった）。
- 既存モード（積立・目標逆算・取り崩し定率）の回帰、コンソールエラー無しを確認。
- 「年」「歳」「%」の手入力をプルダウン（select）に変更した（手入力の手間を減らす目的）。対象：運用期間（1〜40年）、取り崩し開始年齢（50〜80歳）、何歳まで（70〜110歳）、取り崩し期間（1〜60年）、想定利回り（−20〜20%）、利回りの誤差（0〜10%）、インフレ率（0〜10%）、毎年の取り崩し率（0〜20%）。％系は0.5%刻み、年/歳系は1刻み。金額系（万円）と年金受給開始年齢スライダーは対象外（据え置き）。
  - HTML側は `<select data-min data-max data-step data-unit data-default>` の空要素として配置し、JSの `populateNumericSelects()` が起動時に `<option>` を自動生成（ラベルに単位を含める、例「65歳」「5%」）。既存の `.unit` 外側スパンは廃止し `.select-wrap` という新しいラッパーで見た目を `.input-wrap` と揃えた。
  - 既存の `inputs` オブジェクト・`saveState`/`loadState`・`update()` の呼び出しは id 参照のみで変更不要（select の `.value` は input と同じ文字列で扱えるため）。selectの `input` イベントで保存・再計算が発火することを確認。
  - ブラウザ検証：各selectのオプション数・上限下限ラベル、0.5%刻み値（5.5%・3.5%等）の反映・保存・リロード後の復元、積立/目標逆算/取り崩し（定額・定率）全モードでの表示・動作、コンソールエラー無しを確認。
